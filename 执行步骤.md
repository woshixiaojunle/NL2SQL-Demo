## 技术栈确认

| 环节 | 技术选型 | 说明 |
|------|---------|------|
| Embedding 模型 | DashScope `qwen3-vl-embedding` | 已在 init_table_embbeding.py 中使用，保持一致 |

| 对话入口 | 命令行 REPL | 先跑通 CLI，后续可包一层 FastAPI / Streamlit |

## 整体流程图

```
用户提问
   │
   ▼
[Node 1] embed_query          ← DashScope 向量化用户问题
   │
   ▼
[Node 2] retrieve_tables      ← 余弦相似度 TopN，低于阈值走 no_match 分支
   │
   ├─── 相似度 OK ──────────────────────────────────────────────┐
   │                                                            ▼
   │                                               [Node 3] build_db_schema
   │                                                            │
   │                                               [Node 4] build_prompt
   │                                                            │
   │                                               [Node 5] generate_sql
   │                                                            │
   │                                               [Node 6] execute_sql
   │                                                     │          │
   │                                                  成功        出错(≤2次重试)
   │                                                     │          └──► build_prompt(带错误)
   │                                               [Node 7] format_result
   │                                                            │
   │                                                         输出结果
   │
   └─── no_match ──► 提示用户重新描述 ──► 等待输入
                          │
                    update_state 打补丁
                    (更新 question 字段)
                          │
                    从原 checkpoint fork
                    重新执行 embed_query
```

## MemorySaver + update_state 机制

### 为什么用 MemorySaver

- 每次 `graph.invoke` 自动保存 checkpoint（含完整 GraphState）
- 多轮对话保留上下文，用户可以追问"上次那个查询加个时间过滤"
- `no_match` 时不销毁 thread，用户重新输入后可以在原 thread 上继续

### no_match 处理流程

```python
# 1. 第一次执行，retrieve_tables 发现无匹配
config = {"configurable": {"thread_id": "session_001"}}
result = graph.invoke({"question": "查一下库存", "retry_count": 0}, config)
# result["match_status"] == "no_match"，提示用户

# 2. 用户重新输入，直接用同一个 thread_id 重新 invoke
#    MemorySaver 会在同一 thread 上追加新的 checkpoint，上下文不丢失
new_question = input("重新输入: ").strip()
result = graph.invoke(
    {"question": new_question, "retry_count": 0, "match_status": "", "exec_status": ""},
    config   # 同一个 config / thread_id
)

# ❌ 错误做法（as_node="embed_query" 会跳过 embed_query 本身，新问题不会被向量化）
# new_config = graph.update_state(config, patch, as_node="embed_query")
```

### update_state 的正确使用场景

`update_state` 适合在 graph **暂停**（interrupt）后修正某个字段再继续，例如：

```python
# 用 interrupt_before 在 execute_sql 前暂停，让用户确认 SQL
graph = graph_builder.compile(checkpointer=memory, interrupt_before=["execute_sql"])
result = graph.invoke({"question": question}, config)
print("即将执行 SQL：", result["sql"])
confirm = input("确认执行？(y/n): ")
if confirm == "y":
    # 不修改状态，直接继续
    result = graph.invoke(None, config)
else:
    # 修改 sql 字段后继续
    graph.update_state(config, {"sql": "SELECT ..."}, as_node="execute_sql")
    result = graph.invoke(None, config)
```

### SQL 执行出错重试

```python
# execute_sql 节点出错时写入 error 字段，exec_status = "error"
# 条件边：exec_status == "error" 且 retry_count <= 2 → 回到 build_prompt
# build_prompt 检测到 error 字段，将错误信息追加进本次 messages（不累积历史）
# LLM 看到错误后自动修正 SQL
# retry_count 达到上限 → exec_status = "max_retry" → 走向 format_result 输出错误提示
```

## GraphState 数据结构

```python
class GraphState(TypedDict):
    question: str           # 用户原始问题
    query_vector: list      # 问题向量
    matched_tables: list    # 匹配到的 core_table 记录（含相似度分数）
    db_schema: str          # 拼装好的 schema 文本
    messages: list          # 本轮发给 LLM 的消息（每次 build_prompt 重新构建，不累积）
    sql: str                # LLM 生成的 SQL
    result: list[dict]      # 查询结果
    error: str              # 执行错误信息
    retry_count: int        # SQL 重试次数（最多 2，即 retry_count <= 2）
    match_status: str       # 表匹配状态：ok / no_match
    exec_status: str        # SQL 执行状态：pending / success / error / max_retry
```

> `messages` 每轮重新构建，避免多轮累积撑爆 LLM context window。
> 多轮上下文（历史问题）如需保留，单独维护 `history: list[str]` 字段。

## 各节点详细说明

### Node 1 — embed_query
- 调用 `embedding_str(question)` → `query_vector`

### Node 2 — retrieve_tables（含路由）
- 从 `core_table` 读所有 `checked=TRUE` 记录
- `ast.literal_eval` 还原 embedding 字段
- numpy 计算余弦相似度，取 Top 3
- **相似度阈值 0.5**（可配置）
  - 全部低于阈值 → `status = "no_match"`，走条件边到 `__end__`
  - 有匹配 → `status = "ok"`，继续下一节点

### Node 3 — build_db_schema
- 按 `table_id` 查 `core_field`（`field_index` 排序）
- 拼装 schema：
```sql
-- 订单主表
CREATE TABLE orders (
  order_id varchar(32),  -- 订单号
  customer_name varchar(100),  -- 客户姓名
  ...
);
```

### Node 4 — build_prompt
- System：你是 PostgreSQL SQL 专家，只输出 SQL，不要解释
- 若 `error` 不为空，追加：`上次生成的 SQL 执行报错：{error}，请修正`
- Human：`{db_schema}\n\n用户问题：{question}`

### Node 5 — generate_sql
- `llm.invoke(messages)` → 正则提取 SQL（匹配 ```sql...``` 或纯文本）

### Node 6 — execute_sql（含路由）
- `execute_query(sql)` 执行
- 成功 → `status = "done"`
- 失败且 `retry_count < 2` → `retry_count += 1`，`status = "sql_error"`，回到 Node 4
- 失败且已达上限 → `status = "done"`，result 为空，error 保留

### Node 7 — format_result
- tabulate 打印 Markdown 表格
- 同时输出执行的 SQL（调试用）

## 条件边定义

```python
def route_after_retrieve(state: GraphState) -> str:
    return "build_db_schema" if state["status"] == "ok" else "__end__"

def route_after_execute(state: GraphState) -> str:
    if state["status"] == "sql_error" and state["retry_count"] < 2:
        return "build_prompt"
    return "format_result"
```

## 多轮 REPL 伪代码

```python
memory = MemorySaver()
graph = graph_builder.compile(checkpointer=memory)
thread_id = "session_001"
config = {"configurable": {"thread_id": thread_id}}

while True:
    question = input("请输入问题（q退出）: ").strip()
    if question == "q":
        break

    result = graph.invoke({"question": question, "retry_count": 0}, config)

    if result["status"] == "no_match":
        print("⚠️  未找到相关表，请换个描述方式重新输入")
        new_question = input("重新输入: ").strip()
        # 在原 checkpoint 上打补丁，从 embed_query 重跑
        new_config = graph.update_state(
            config,
            {"question": new_question, "retry_count": 0, "status": "retry"},
            as_node="embed_query"
        )
        result = graph.invoke(None, new_config)
```
